Resource Profile: VA MHV PHR HDR CBC Report

Official URL: https://johnmoehrke.github.io/MHV-PHR/StructureDefinition/VA.MHV.PHR.cbcReport Version: 0.1.8-beta
Standards status: Trial-use Maturity Level: 0 Computable Name: MHVcbcReport

A profile showing how labTestPromise is mapped into a FHIR DiagnosticReport, Observation, and Specimen. Where the labTestPromise only has labSubscript of CH (presumed CBC).

  • LOINC#58410-2 ‘CBC panel - Blood by Automated count’

The given example aligns with the Lab.xsd schema.

The labTestPromises is mapped onto this FHIR DiagnosticReport for CBC laboratory reporting. The mapping to HDR labTestPromise

The labTestPromises.labTests are each recorded as a FHIR Observation for CBC result that is contained in the DiagnosticReport. The map to HDR cbcTestPromise.

The labTestPromises.specimen is mapped into a FHIR Specimen resource that is contained in the DiagnosticReport. The map to HDR Specimen.

The use of contained means that we do not need to de-duplicate the lab tests or specimen.

This profile is based on US-Core DiagnosticReport profile for Laboratory Results Reporting and lab Observations. Following the US-Core example for CBC
